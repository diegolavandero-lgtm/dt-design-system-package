Context-aware confirm modal copy #context-modal RULE

The confirmation modal copy should be specific and consequence-led. Title = the action. Body = the impact. Copy must reference the exact item being affected, not use generic "confirm" language.

✓ Do — specific + consequence

Disable the "Delivered" notification?

Customers will no longer be notified when an order reaches "Delivered". You can turn it back on at any time.

✗ Don't — generic + repetitive

Are you sure want to disable this option?

Info vs Warning banner — semantic rule #banner-rule RULE

The .bn banner component has four intents. The amber (warning) banner was being misused for informational messages in production. Rule: amber = something requires the user's attention NOW. Blue = neutral system information.

✓ Blue — informational
To enable SMS or WhatsApp notifications, contact support@dispatchtrack.com. You can still design the message now.
✓ Amber — action required

Paused channel semantics #paused-semantics RULE

The toggle in a notification row operates at the channel level, not the notification level. Three distinct states must be visually communicated in any settings UI where notifications can be configured per channel.

StateToggle visualMeaningSub-label
Active
Channel is sending Original notification name
Paused (1 of N)
This channel manually paused · Others still active "Email paused · SMS still active"
All paused
No channel sending — notification silent "No channel active · Notifications not being sent"
Not configured
No notification created yet "+ Add notification" (CTA, not status)
Locked
Channel not enabled for account Account-level channel status
Key rule: the amber --o5 ⏸ toggle is only used when a user intentionally disables a channel that was previously active. It must never appear for channels that are locked or unconfigured — those use gray.
