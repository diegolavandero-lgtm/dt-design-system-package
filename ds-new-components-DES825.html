✓ Do

One toggle per alert type. Show status chip (Active / Paused) in the collapsed row. Form appears only on expand. Disable all inputs when toggle is off.

Header Action Bar #header-actions RULE

Rule for account configuration pages: primary actions (Save changes, Cancel) always live in the page header, right-aligned alongside secondary actions. All buttons use the blue outline style by default; only the primary action uses the filled style. No icons on any header button.

Alerts

✓ Do

Actions always at header level, same line as title. Order: secondary (outline) → primary (filled), left to right.

✗ Don't

No icons on header buttons. No sticky save bar at the bottom — it falls out of viewport on long pages and is inconsistent with the global pattern.

Chip Tag Input #chip-input EXTENDED

Extension of the existing chip pattern for multi-select inputs. Chips are removable with ×, generated from a checkbox dropdown, and display full labels (no truncation). A chevron indicator signals the dropdown affordance. Applies to statuses, substatuses, labels, and any multi-selectable entity.

Substatuses — production values.chip · .ci-wrap
Delivered to customer Wrong address Customer absent Product rejected Damaged product
✓ Do

Full labels in chips — never truncate. Chevron always visible to signal the dropdown. font-weight: 400 (regular) for chip text.

✗ Don't

Don't truncate chip labels with "…" — the user can't read what's selected. Don't use bold text in chips — it competes with section labels.

Alert Status Chip #status-chip EXTENDED

Extension of the existing status pill pattern for operational alert states. Two variants: Active (green) and Paused (grey). Always used alongside the toggle for semantic redundancy (color + text label).

Both states.sc.on · .sc.off
Active
Toggle ON
Paused
Toggle OFF
✓ Do

Keep chip and toggle in sync. When toggle changes, chip updates immediately. Always include a text label alongside the color dot.

✗ Don't

Don't use red for "Paused" — red implies error. Grey communicates "inactive" without alarming. Don't show chip without toggle, or toggle without chip.

Info Tooltip (ⓘ) #info-tooltip NEW

Inline information icon that shows a custom tooltip on hover with additional context about a field. Uses the "i" in circle icon (not a question mark). Tooltip appears instantly on hover, follows the cursor, and disappears on mouse-out — bypassing the native browser delay.

Icon states + tooltip.help-icon · .tip-box
Default
Hover
Alert fires when the post-delivery customer rating is at or below this score. Useful for flagging poor delivery experiences.
✓ Do

Use ⓘ for fields with non-obvious logic. Tooltip text should explain system behavior, not repeat the field label. Keep under 40 words.

✗ Don't

Don't use ? (question mark) — in the DS the question mark implies external help. ⓘ is for inline context. Don't rely on native title attribute — it has an unpredictable delay and can't be styled.
