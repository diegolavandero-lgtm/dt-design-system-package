Alert Settings Row alert-row NEW

Patrón de fila accordion para páginas de configuración de cuenta. Combina nombre + descripción, chip de estado, toggle de activación y cuerpo expandible con formulario inline. Diseñado para Settings screens con múltiples configuraciones independientes de baja frecuencia de edición.

Chip Tag Input chip-input EXTENDED

Extensión del patrón de chips existente para inputs de selección múltiple. Los chips son removibles con ×, se generan desde un dropdown con checkboxes y muestran el label completo (sin truncado). Aplica a selects de estados, subestados, etiquetas y cualquier entidad multi-seleccionable.

Chip Tag Input — inline removable con dropdown
chip · multi-select-box · ms-dropdown

Subestados seleccionados

Cita Cerrada Pedido Asignado Fuera de Ruta Entrega Exitosa Local Cerrado + 7 más…
✓ Hacer

Labels completos en los chips. Si hay más de 8, agrupa con "+ N más" que expande.

✗ No hacer

No truncar labels con "…" dentro del chip. El usuario no puede leer qué está seleccionado.

Sticky Save Bar save-bar NEW

Barra de acción pegada al fondo de páginas de configuración que requieren guardado explícito (no auto-save). Aparece al detectar cambios no guardados (dirty state) y desaparece al guardar o cancelar. Incluye mensaje de "cambios sin guardar" + botones Cancelar y Guardar.

Sticky Save Bar — dirty state + acciones
save-bar · save-bar.visible

Vista del save bar (simulado)

Tienes cambios sin guardar
✓ Hacer

Usar el save bar en todas las páginas de configuración que modifican settings de cuenta. Mostrar solo cuando hay cambios (dirty state).

✗ No hacer

No usar en páginas con auto-save (toggles de feature flags, preferencias de usuario). No reemplazar por botón flotante o toast.

Alert Status Chip status-chip EXTENDED

Extensión del patrón de status pill existente para representar el estado de alertas operacionales. Tiene dos variantes: Activa (verde) y Pausada (gris). Se usa siempre junto al toggle para dar redundancia semántica (color + texto).

Alert Status Chip — Activa / Pausada
status-chip.on · status-chip.off
Activa
Toggle ON
Pausada
Toggle OFF
✓ Hacer

Sincronizar chip y toggle. Cuando el toggle cambia, el chip debe actualizarse inmediatamente. Siempre incluir label de texto además del dot.

✗ No hacer

No usar rojo para "Pausada" — rojo implica error. El gris comunica "inactivo/deshabilitado" sin alarmar.
